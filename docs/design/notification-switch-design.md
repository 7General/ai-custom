# 通知开关功能（跨设备同步）- 技术设计文档

## 文档信息
- 版本：v1.0
- 日期：2026-06-11
- 状态：已批准

---

## 1. 概述

### 1.1 背景
用户在多设备使用 App 时，通知开关设置无法跨设备同步，需重复配置。

### 1.2 目标
提供 Push 推送通知开关，用户修改后自动同步至服务端，实现多设备统一生效。

### 1.3 范围
- 仅针对已登录用户，未登录用户无入口
- 通知类型：Push 推送（单一类型，后端配置）
- 不包含系统级推送权限申请

---

## 2. 需求分析

### 2.1 功能需求

| # | 需求 | 说明 |
|---|------|------|
| F1 | 通知开关设置 | 用户可开启/关闭 Push 推送通知 |
| F2 | 跨设备同步 | 修改后同步至服务端，其他设备进入页面时读取最新状态 |
| F3 | 本地即时生效 | 开关变更后当前设备立即响应，不等待网络结果 |
| F4 | 离线处理 | 无网络时允许本地修改，网络恢复后自动同步 |
| F5 | 冲突处理 | 多设备同时修改时，以服务端 updated_at 最新值为准 |

### 2.2 用户故事

**Story 1：基础开关**
> 作为 已登录用户
> 我希望 能开关 Push 推送通知
> 以便  控制是否接收推送消息

**Story 2：跨设备同步**
> 作为 多设备用户
> 我希望 在设备A关闭推送后，设备B上也同步关闭
> 以便  不需要在每台设备重复设置

**Story 3：离线修改**
> 作为 网络不稳定的用户
> 我希望 没有网络时也能修改开关，恢复联网后自动同步
> 以便  设置操作不被网络状态阻断

### 2.3 验收条件（EARS格式）

**AC1 - 开关修改**
```
GIVEN 用户在通知设置页
WHEN  点击 Push 推送开关
THEN  UI 立即响应（toggle 状态变化）
AND   变更请求异步上传服务端
```

**AC2 - 跨设备同步**
```
GIVEN 用户在设备A将 Push 推送关闭
WHEN  用户在设备B进入通知设置页
THEN  Push 推送显示为关闭状态
```

**AC3 - 进入页面同步**
```
GIVEN 用户进入通知设置页
WHEN  页面加载
THEN  自动拉取服务端最新配置并展示
```

**AC4 - 离线修改**
```
GIVEN 用户设备无网络
WHEN  修改通知开关
THEN  本地立即生效
AND   网络恢复后自动上传，服务端以最新 updated_at 为准
```

**AC5 - 首次登录新设备**
```
GIVEN 用户首次在新设备登录
WHEN  进入通知设置页
THEN  从服务端拉取该账号的通知开关配置，不使用设备默认值
```

## 3. 接口设计

### 3.1 接口列表

| # | 方法 | 路径 | 说明 |
|---|------|------|------|
| 1 | GET | /api/v1/notification/settings | 获取用户通知开关状态 |
| 2 | PUT | /api/v1/notification/settings | 更新通知开关状态 |

### 3.2 接口详情

#### 接口一：获取用户通知开关状态

**GET** `/api/v1/notification/settings`

请求头
```
Authorization: Bearer {token}
```

响应示例
```json
{
  "code": 200,
  "data": {
    "push_enabled": true,
    "updated_at": "2026-06-11T10:00:00Z"
  }
}
```

---

#### 接口二：更新通知开关状态

**PUT** `/api/v1/notification/settings`

请求头
```
Authorization: Bearer {token}
Content-Type: application/json
```

请求体
```json
{
  "push_enabled": false
}
```

响应示例
```json
{
  "code": 200,
  "data": {
    "push_enabled": false,
    "updated_at": "2026-06-11T10:30:00Z"
  }
}
```

### 3.3 错误码

| code | 说明 |
|------|------|
| 200 | 成功 |
| 401 | 未登录 / token 失效 |
| 500 | 服务端异常 |

---

## 4. 客户端实现要点

| 要点 | 说明 |
|------|------|
| 同步时机 | 进入通知设置页时触发 GET 接口 |
| 本地存储 | UserDefaults 缓存开关状态，保证即时响应 |
| 防抖处理 | 用户快速连续点击开关时，客户端做防抖，避免重复请求 |
| 离线处理 | 无网络时本地修改生效，网络恢复后自动上传 |
| 冲突策略 | 以服务端返回的 updated_at 最新值为准 |
