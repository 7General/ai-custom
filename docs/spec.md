# SDD Spec - AB测功能（Feature Flag）

## 1. 组件定位

### 1.1 核心职责

AB测组件负责管理客户端AB测实验配置的生命周期，包括默认值兜底、服务端数据拉取、本地缓存持久化，以及向业务方提供同步读取接口。

### 1.2 输入

- 预配置的默认值集合（键值对）
- 服务端返回的AB测配置数据
- 业务方的读取请求（实验key）

### 1.3 输出

- 实验值（根据key返回对应的value）
- 数据加载状态（成功/失败）

### 1.4 边界

- **包含**：默认值管理、网络数据拉取、本地缓存读写、对外读取接口
- **不包含**：用户分组逻辑、数据分析与埋点上报、后台管理系统

---

## 2. 领域术语

| 术语 | 定义 |
|------|------|
| 实验（Experiment） | 一个AB测配置项，由实验名称（key）和实验值（value）组成 |
| 实验值（Value） | 某个实验在当前用户分组下的取值，类型为字符串 |
| 默认值（Default） | 客户端预配置的实验初始值，网络和缓存数据均不可用时使用 |
| 缓存数据（Cache） | 最近一次网络获取成功后持久化到本地的AB测配置数据 |
| 数据源优先级（Priority） | 数据加载的优先级顺序：缓存 > 默认值 |

---

## 3. 角色与边界

### 3.1 核心角色

| 角色 | 职责 |
|------|------|
| AB测管理器 | 管理AB测配置的完整生命周期：初始化、加载、缓存、读取 |
| 业务调用方 | 通过管理器读取AB测实验值，决定业务行为 |

### 3.2 外部系统

| 外部系统 | 交互方式 | 说明 |
|----------|----------|------|
| AB测服务端 | 网络请求 | 提供AB测配置查询能力，返回键值对格式的实验数据 |
| 本地存储 | 文件读写 | 持久化AB测缓存数据，使用项目已有的存储机制 |

### 3.3 交互上下文

- 业务调用方在任意时机通过实验key同步读取实验值
- AB测管理器在初始化时加载缓存或默认值，并异步拉取最新数据
- 网络获取成功后自动更新内存和缓存，无需业务方干预

## 4. DFX约束

### 4.1 性能

| 约束 | 指标 |
|------|------|
| 读取响应时间 | 同步读取，< 1ms（内存操作） |
| 网络请求超时 | <= 10s |
| 缓存读取耗时 | < 50ms |
| 内存占用 | AB测数据内存占用 < 1MB |

### 4.2 可靠性

| 约束 | 指标 |
|------|------|
| 网络失败降级 | 网络不可用时100%使用缓存或默认值，不影响业务正常运行 |
| 缓存解析失败降级 | 缓存数据损坏或格式不兼容时，回退到默认值 |
| 线程安全 | 多线程并发读取不崩溃、不错乱 |

### 4.3 可维护性

| 约束 | 指标 |
|------|------|
| 接口简洁性 | 业务方接入仅需调用1-2个方法 |
| 默认值可配置 | 默认值集合支持外部注入，无需修改组件源码 |

### 4.4 兼容性

| 约束 | 指标 |
|------|------|
| iOS版本 | 与MSCustomAi组件库最低支持版本一致 |
| 依赖 | 仅使用项目已有依赖 |

## 5. 核心能力

### 5.1 能力一：默认值管理

**业务规则：**
- BR-001: 默认值在管理器初始化时注入，格式为键值对集合
- BR-002: 未配置默认值的实验，读取时返回空字符串
- BR-003: 默认值优先级低于缓存数据，仅在无缓存时作为兜底

**验收条件（EARS格式）：**
- When 管理器初始化时传入默认值集合，the AB测管理器 shall 将默认值加载到内存中
- When 读取一个未配置默认值且无缓存的实验key，the AB测管理器 shall 返回空字符串
- While 本地无缓存数据，the AB测管理器 shall 使用默认值作为读取结果

**异常场景：**
- If 传入的默认值集合为空，the AB测管理器 shall 正常初始化，所有实验读取返回空字符串

### 5.2 能力二：本地缓存管理

**业务规则：**
- BR-004: 网络获取成功后，自动将完整的AB测配置数据持久化到本地
- BR-005: 启动时优先从本地缓存加载数据，缓存数据优先级高于默认值
- BR-006: 新的网络数据覆盖旧的缓存数据
- BR-007: 缓存数据跟随项目已有的缓存清理机制

**验收条件（EARS格式）：**
- When 网络获取AB测数据成功，the AB测管理器 shall 将数据持久化到本地缓存
- When 管理器初始化且本地有缓存数据，the AB测管理器 shall 优先加载缓存数据到内存
- When 网络获取到新数据，the AB测管理器 shall 覆盖更新本地缓存
- If 缓存数据读取失败或格式损坏，the AB测管理器 shall 回退使用默认值，不崩溃

### 5.3 能力三：网络数据获取

**业务规则：**
- BR-008: 网络请求异步执行，不阻塞业务方的同步读取操作
- BR-009: 网络请求成功后，同时更新内存数据和本地缓存
- BR-010: 网络请求失败时，保持当前数据不变（缓存或默认值）

**验收条件（EARS格式）：**
- When 触发AB测数据获取，the AB测管理器 shall 异步发起网络请求，不阻塞调用线程
- When 网络请求成功返回数据，the AB测管理器 shall 更新内存中的AB测数据
- If 网络请求失败（超时、服务端错误、无网络），the AB测管理器 shall 保持使用当前已有数据，不影响业务读取

**交互流程：**
1. 业务方或系统触发数据获取
2. 管理器异步发起网络请求
3. 请求成功：更新内存 -> 更新缓存
4. 请求失败：保持现状，不做任何变更

### 5.4 能力四：数据加载优先级

**业务规则：**
- BR-011: 数据加载优先级为 缓存数据 > 默认值
- BR-012: 网络获取成功后，内存中的数据立即更新为最新值
- BR-013: 同一个实验key在数据更新后，后续读取返回最新值

**验收条件（EARS格式）：**
- While 本地有缓存数据，the AB测管理器 shall 优先使用缓存数据而非默认值
- While 本地无缓存数据，the AB测管理器 shall 使用默认值
- When 网络获取成功后，the AB测管理器 shall 后续读取返回最新的网络数据

### 5.5 能力五：对外读取接口

**业务规则：**
- BR-014: 通过实验名称（key）同步读取实验值
- BR-015: 读取操作为同步操作，从内存直接返回，无IO等待
- BR-016: 支持业务方传入自定义fallback值，当key不存在时返回该fallback
- BR-017: 未传入fallback时，key不存在返回空字符串

**验收条件（EARS格式）：**
- The AB测管理器 shall 提供通过实验key同步读取实验值的能力
- When 读取一个存在的实验key，the AB测管理器 shall 返回该实验的当前值
- When 读取一个不存在的实验key且未指定fallback，the AB测管理器 shall 返回空字符串
- When 读取一个不存在的实验key且指定了fallback值，the AB测管理器 shall 返回该fallback值
- The 读取接口 shall 在任意线程调用时均能安全返回结果

## 6. 数据约束

### 6.1 领域对象逻辑约束

| 对象 | 约束 |
|------|------|
| 实验key | 非空字符串，作为实验的唯一标识 |
| 实验value | 字符串类型，允许为空字符串 |
| 默认值集合 | 键值对集合，key为实验名称，value为默认值字符串 |
| 缓存数据 | 与服务端返回的完整AB测配置数据结构一致，支持序列化/反序列化 |

### 6.2 数据生命周期约束

| 约束 | 说明 |
|------|------|
| 缓存持久性 | 缓存数据持久化存储，App重启后仍存在 |
| 缓存清理 | 遵循项目已有的缓存清理机制（如用户登出时的清理策略） |
| 默认值生命周期 | 默认值随管理器初始化注入，App生命周期内不变 |
