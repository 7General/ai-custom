# 质量阶段

## 阶段概述

质量阶段通过系统化的质量检查流程，确保代码满足质量标准和安全要求。通过静态分析、安全扫描、性能分析等多维度检查，在代码合并前发现并修复潜在问题，实现"质量左移"。


## 前置规范参考

| 规范名称  | Reference |
|---------------|  ------- |
|编码通用规范(必须)| 	`code-skill/SKILL.md`|
|SnapKit 约束规范（代码包含 UI 布局时）| 	`snapkit-skill/SKILL.md`|
|网络请求规范（代码包含网络请求时）| 	`request-send-skill/SKILL.md`|
|组件结构 |	`rules/组件结构.md`|
|编码规范 |	`rules/编码规范.md`|

## 核心活动

### 1. 代码规范检查

**目标**：确保代码符合团队编码规范

**检查内容**：
- 代码格式化
- 命名规范
- 编码规范
- 注释规范
- 代码结构
- 组件结构


### 2. 静态分析

**目标**：在不运行代码的情况下发现潜在问题

**分析内容**：
- 代码复杂度
- 代码重复
- 潜在Bug
- 代码异味
- 技术债务


### 3. 性能分析

**目标**：识别性能瓶颈和优化机会

**分析内容**：
- 内存泄漏分析
- 资源泄漏检测


## 质量检查流程

### 标准质量检查流程

```
代码提交
    ↓
代码规范检查
    ├─ 代码格式化
    ├─ 命名规范
    ├─ 编码规范
    ├─ 注释规范
    ├─ 代码结构
    └─ 组件结构
    ↓
静态分析
    ├─ 复杂度分析
    ├─ 重复代码检测
    └─ 潜在Bug检测
    ↓
性能分析
    ├─ 内存泄漏分析
    └─ 资源泄漏检测
    ↓
质量门禁判断
    ├─ 各项指标达标？
    ├─ YES → 允许合并
    └─ NO  → 阻止合并，返回问题
```
