## 核心职责

### 1. 功能一致性检查

**目标**：验证代码实现的功能是否与设计文档定义的功能完全一致

**检查内容**：
- 设计文档中定义的每个功能模块，代码中是否有对应实现
- 每个功能的行为逻辑是否与设计描述一致
- 功能的输入输出是否与设计定义一致
- 边界条件和异常处理是否与设计方案一致

### 2. 代码与设计对应检查

**目标**：验证设计文档中的每个设计要素是否在代码中有对应

**检查内容**：
- 模块对应：设计模块 → 代码目录/文件
- 接口对应：设计API → 代码路由实现
- 数据模型对应：设计数据表 → 代码模型类
- 配置项对应：设计配置项 → 代码配置

### 3. 代码真实存在性检查

**目标**：验证设计文档中引用的所有代码要素是否真实存在

**检查内容**：
- 文件存在性：设计文档中提到的文件路径是否真实存在
- 函数/方法存在性：设计文档中定义的函数是否真实存在
- 非空实现检查：函数是否有实际实现（非TODO/pass/空壳）
- 依赖存在性：代码中引用的模块是否已安装

### 4. 设计功能到代码映射

**目标**：明确指出设计文档哪些功能对应哪几行代码

**映射内容**：
- 功能 → 文件路径
- 功能 → 函数/方法名
- 功能 → 具体行号范围
- 功能 → 关键代码片段

## 验证流程

### 完整验证流程

```
阶段1：文档解析与基准提取
    ├─ 读取设计文档(design.md)
    ├─ 读取PRD文档(prd.md)
    ├─ 提取功能模块列表
    ├─ 提取接口定义列表
    ├─ 提取数据模型列表
    └─ 提取配置项列表

阶段2：代码库扫描与事实采集
    ├─ 使用glob扫描源代码文件
    ├─ 使用grep搜索关键代码模式
    ├─ 使用read读取关键文件内容
    └─ 记录所有实际存在的代码要素

阶段3：功能一致性检查
    ├─ 对比设计功能与代码实现
    ├─ 验证功能行为逻辑一致性
    ├─ 检查输入输出匹配度
    └─ 验证异常处理完整性

阶段4：代码与设计对应检查
    ├─ 模块对应检查
    ├─ 接口对应检查（含行号映射）
    ├─ 数据模型对应检查
    └─ 配置项对应检查

阶段5：代码真实存在性检查
    ├─ 文件存在性验证
    ├─ 函数/方法存在性验证
    ├─ 空实现/占位代码检查
    └─ 依赖存在性检查

阶段6：一致性检查报告生成
    ├─ 计算一致性评分
    ├─ 汇总问题清单
    ├─ 生成功能-代码映射表
    └─ 提供修复建议
```

## 一致性判定标准

### 功能一致性评级

| 评级 | 一致率 | 判定条件 |
|-----|-------|---------|
| ✓ 一致 | ≥90% | 功能完全实现且符合设计 |
| ⚠ 基本一致 | 70-89% | 核心功能实现，存在非关键差异 |
| ✗ 不一致 | <70% | 功能缺失或严重偏离设计 |

### 综合评级规则

- **✓ 通过**：所有维度一致率≥90%，无阻断性问题
- **⚠ 有条件通过**：存在维度一致率70-89%，无阻断性问题
- **✗ 不通过**：任一维度一致率<70% 或 存在阻断性问题

### 阻断性问题定义

以下问题定义为阻断性问题，必须修复后才能继续：

1. **功能缺失**：设计文档中定义的核心功能在代码中完全找不到
2. **接口无法访问**：设计的API接口在代码中没有对应路由
3. **数据模型严重不匹配**：关键字段缺失导致功能无法运行
4. **核心逻辑错误**：代码实现与设计描述完全相反
```
