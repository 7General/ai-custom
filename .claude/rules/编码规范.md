## 编码规范


### 1. 代码命名
- [ ] 所有新创建的文件必须以 `OPB` 开头（如 `OPBHomeViewController`、`OPBOrderDetailView`）
- [ ] ViewController：名称 + VC 后缀（如 `HomeViewController`）
- [ ] 自定义 View：描述性名称 + View 后缀
- [ ] 工具类：LS 前缀 + 用途 + Tool/Helper
- [ ] MARK 注释**必须中文**（如 `// MARK: - 属性`、`// MARK: - 生命周期`）
- [ ] 在声明懒加载变量用闭包时，里面的临时对象用`it`
- [ ] 不能用下划线`_`开头的变量
- [ ] 不能重复代码定义
- [ ] 不能以用下划线`_`开头的方法
- [ ] 声明的普通变量要在 `viewDidLoad()`前声明
- [ ] 声明的方法和懒加载的变量要在 `viewDidLoad()`后声明

### 2.路由跳转

使用`OPRouter`-pod库，做跳转工具，如果没有导入，则`import OPRouter`
- [ ] `OPNewRouterManager.registerRouterItem(urlKey, className:)` — 注册
- [ ] `OPNewRouterManager.openURL(url, userInfo:)` — push 跳转
- [ ] `OPNewRouterManager.modelURL(url, userInfo:)` — present 模态跳转
- [ ] Route keys are `opay://` or `opaypos://` scheme URLs 命名前缀

### 3.UIViewController结构和约束

- [ ] 需要继承`OPBUIViewController`.并添加方法 `setupUI()`, `setupLayout()`, `setupAction()`, `setupStyle()`
- [ ] 并添加`deinit`代码
```swift
deinit {
    NotificationCenter.default.removeObserver(self)
    debugPrint("[当前控制前名称]======dealloc")
    }
```

### 4.View相关结构和约束

- [ ] 当需要自定义的`UIView`时，需要继承`OPBBaseView`.在子类重写 `setupUI()`, `setupLayout()`, `setupAction()`, `setupStyle()`
- [ ] 并添加`deinit`代码
```swift
deinit {
    NotificationCenter.default.removeObserver(self)
    debugPrint("[当前文件前名称]======dealloc")
    }
```

### 5.主题颜色适配

适配暗黑用 **SwiftTheme**:
- [ ] `theme_backgroundColor`, `theme_textColor`, `theme_image` for themed properties
- [ ] `OPBThemeHelper` provides theme color pickers
- [ ] `OPThemeManager.isDark()` / `OPThemeManager.setupTheme()` for mode control

### 6.数据与缓存

- [ ] **`OPBStorageHelper`** — file-based persistent storage (documents directory), survives logout
- [ ] **`UserDefaults`** — feature flags, guide states; partially cleared on logout in `OPBLoginKit+cache.swift`
- [ ] When adding new `UserDefaults` keys that should survive logout, add them to the whitelist in `OPBLoginKit+cache.__clearCache()`

### 7.埋点事件

- [ ] `OPBEventLogHelper.__addEventPrefixLog(eventName:, attributes:)` — 默认使用，如果没有attributes，可不传
